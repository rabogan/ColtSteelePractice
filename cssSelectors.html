Basic Selectors: Universal and Element Selectors

  • First covered: the universal selector * is shown, but its use is not recommended. It basically tries to change all elements on the page, so doesn't really meet our needs very often
  • The element selector (e.g. button/h1 etc...) will changes all elements of that type
  • The obvious example is something like a h1{ ... } - all h1 elements will have the ... attributes adjusted
  • However, using a ,(comma) to separate elements is also acceptable. For example, h1+h2{...} means h1 AND h2 will have their attributes adjusted. Later on in this section, you'll see that ID-types and class-types can be represented in the same way. Using a class "complete" or id "example", the following could also work: .complete+h1{...} #id+h1{...}

ID Selectors

  • We can use the # id to single in on individual elements which use them inline. In my example, I changed the page body's background-color to #f1faee. Both of the button

  • We wanted ONE specific button (the Sign Up button) to have a different color scheme. To do so, I added an id called 'signup' to that specific button element, and then changed the attributes associated with that id name. So, something like #signup{...} is used in the css file
  • One HUGELY important thing: an id of a specific name should only be used once on a single HTML page. We shouldn't overuse id selectors, but there's no need to avoid using them. A handful on a single page seems to be optimum, but class selectors are much more common

The Descendant Selector

  • The SPACE between elements in the css file is very important!
    • h1,h2 {...} : the comma means h1 AND h2
    • h1 h2{...}: the space means any descendant h2 INSIDE OF h1 will have attributes adjusted
  • As another example, we can choose to select all <a>s elements that are nested inside an <li> element. To do so, we can use li a{ ... }
  • So, no li tags would be styled in this...ONLY anchor tags <a> that are inside an <li> tag
  • In the code of this page, one section below has a class 'post' appended to it. All anchor tags within that class post were then changed according to the attributes defined within the class. This uses .post a{...} in the CSS file
  • Finally, a footer a (any a found INSIDE OF (or descending from) footer) was used in CSS to change the color of those particular links

The Combinator (Adjacent Selector/Adjacent Sibling Selected) and Direct Child/Direct Descendant Selector

    • h1,h2 {...} : the comma means h1 AND h2
    • h1 h2{...}: the space means any descendant h2 INSIDE OF h1 will have attributes adjusted
    • h1+h2{...}: the + sign means any h2 immediately preceded by h1 will have its attributes adjusted
    • h1>h2{...} the > sign denotes that any h2 element that is a DIRECT CHILD (as opposed to a non-direct descendant nested deeply inside of) a h1 element will have its attributes adjusted
  • The h1+p {...} in the lecture's example is known as a COMBINATOR. It will ONLY change paragraphs immediately preceded by h1
  • In our page's example, any button that follows input, as well as any button following a h2 has been changed to slightly more garish colors
  • Child/Direct Descendant Selectors are quite easy. They make use of the > sign, as in footer > a, in my CSS file! Essentially, it demands that ONLY direct descendants of a specific selector are looked at. In this page, footer a should yield red links, while footer > a should yield blue ones

The Attribute Selector

Examples

  1. input[type="text"]{...}: This selects all <input> elements where the type attribute is set to text
  2. input[type="password]{...}: This selects all <input> elements where the type attribute is set to password
  3. section[class="post"]{...}: This selects all <input> elements where the class attribute is set to post
  4. a[href="https://www.rabogan.github.io"]{...}: This selects all <a> elements where the href attribute matches the www.rabogan.github.io website
  • A little uncommon, the above allows us to select all elements that contain a SPECIFIC ATTRIBUTE, such as all inputs of type 'password' (let's say you want to change the text color of the password input)
  • There are multiple attribute selectors (most rarely used), such as:
    • element1=[attribute="example"] = used in this way denotes a DIRECT match
    • element1*=[attribute="example"] *= denotes a 'fuzzy' match. So, if a href contains even the word 'goo', that could take you to google, goodbar, gooner4lyfe etc...
    • element1$=[attribute="example"] $= used likes this means that the match for an element element1 must end with example. Most used in websites!
    • ...and many many more!!!

PSEUDO CLASSES

  • A pseudo-class in CSS is a keyword added to a selector that specifies a special state of the selected element
  • Common examples include :hover, :visited, :nth-child(), and :active
  • A common example is to change the color of a button when the mouse is hovering over it! To do this, we need to use a pseudo-class in our CSS file. In my example, I used button:hover{...}, and changed the background-color and color of all buttons that don't have their specificity overridden!
  • So, to change all buttons so that when the mouse is over them, the background-color changes to red and text to white, we use button:hover{...}, where ... is the color changes
  • Another example is this: input[type="radio"]:checked{...} In this, all inputs of type radio that have been checked will have specific attributes adjusted (usually some kind of box-shadow)
  • Finally, :nth-of-type() is very useful. Plugging in a number, e.g. p:nth-of-type(2){...} will simply changed the 2nd p-element among siblings of type p. However, p:nth-of-type(2n){...} will change every 2nd element among siblings of type p

    • Briefly, pseudo-elements

  • A pseudo-element in CSS is a keyword added to a selector that allows you to style a specific part of the selected element(s)
  • Common examples include ::before, ::after, ::selection, ::first-line, and ::first-letter
  • Within this page, my example involves having the first letter of every h2 tag on the page be resized to 36px: h2::first-letter{...}
  • In addition, the ::selection{...} standalone code in my CSS file made it so that most selected text is red and gold
  • The parts that differ are for any text selected with a h2: inspect to see why!

    • Specificity

  • This is quite an easy concept. Basically:
    • ID > Class > Element > Universal (Class includes pseudo-classes, Element includes pseudo-elements)
    • The common tiebreaker is to count the number of each type, with precedence given the highest leftmost type
    • If there are an identical number of classes/ids/pseudo-things/etc..., then the LAST one read as input (i.e. the last one entered in the stylesheet, or the last one of the last-entered stylesheet in the head section) will take precedence!
  • One thing to note is that inline styles have higher specificity than ids! For example, something like <button> style="color:coral"> will trump any number of ids, classes or elements associated with a button in the CSS stylesheet. They're avoided because they can cause serious confusion
  • On the topic of confusion, the !important exception is an example of bad practice. When the !important rule is used on a style declaration, it signals to the browser that the specific declaration should override ANY other declarations. Why is it bad practice? Simply, it makes things difficult to debug, as it breaks the cascading between stylesheets!
